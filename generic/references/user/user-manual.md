# 未来工作厘定助手（通用版）—— 用户说明手册

## 这是什么

一个跨领域的 AI 工作厘定工具。

它不是一个普通的 AI 问答机器人，也不是一个只讲趋势的顾问。
它的作用是：
**带着你把工作拆开看，判断哪些工作会被 AI 接走、哪些会被增强、哪些会升值，并主动帮你从 GitHub 开源社区找到现成的参考实现。**

---

## 适合谁用

### 管理者 / 负责人
- 团队负责人、部门负责人、职能负责人、管理层
- 可以用来：看团队工作未来怎么变、判断哪些值得先重构、找试点方向

### 个体 / 员工
- 普通员工、骨干、基层主管、岗位承担者
- 可以用来：拆自己的工作、判断危险区和升值区、找 AI 落地起点

---

## 最少准备

只要说清楚三件事就能开始：
1. 你想讨论什么工作（一个岗位 / 一项具体工作 / 一个困扰你的痛点）
2. 当前最大的困扰是什么
3. 你是从管理者视角还是个体视角来用

不需要一上来就准备文档或数据。

---

## 建议准备（让判断更稳）

- 当前工作/职责说明（几条文字就行）
- 工作清单（日常做什么 + 周期做什么）
- 典型流程或案例（1-2 个例子）

如果你不知道怎么整理，可以直接用内置的工具表来盘点工作。

---

## 会带你怎么走（6 步）

1. **先看清工作本质** — 先不急着谈 AI，先搞清楚这个工作更像规则型、风险型还是设计型
2. **把工作拆开** — 把大工作拆成不同颗粒度，看清哪些是高频低价值、哪些是低频高价值
3. **判断每类问题怎么处理** — 是需要明确答案、需要看风险、需要诊断原因，还是需要设计方案
4. **判断未来谁来做** — 哪些可以自动接管、哪些 AI 先做一层人再把关、哪些必须人主导
5. **判断 AI 进来的主要价值** — 是更快、更稳、更广，还是帮你看得更深、设计得更好
6. **主动帮你搜开源方案** — 对适合 AI 化的工作，直接帮你去 GitHub 找现成的工具和项目

---

## 最后你会得到什么

### 如果你是管理者
- 工作结构全景图
- 未来变化趋势判断
- 价值重心迁移方向
- AI 落地优先级排序
- 试点建议与跨角色共创议题

### 如果你是个体员工
- 自己的工作拆解图
- 危险区 / 升值区 / 可试区
- 能力升级方向
- AI 起步建议
- 下一步动作建议

### 如果发现适合 AI 化的工作
- 从 GitHub 开源社区检索到的成熟参考实现
- 每个推荐项目的映射说明、成熟度评估、采纳难度、建议路径

---

## 它和普通 AI 顾问有什么不同

普通 AI 顾问更像是回答问题 —— 你问什么，它答什么。

这个工具更像是：
**带着你把工作拆开、看清、判断，并收敛出未来方向和具体路径。**

所以它不是只给你观点，而是给你：
- **工作结构**（你的工作到底由什么组成）
- **价值判断**（哪些会变、往哪变）
- **落地边界**（现在能不能做、值不值得做）
- **参考实现**（GitHub 上有没有现成的工具可以用）
- **行动建议**（具体下一步做什么）

---

## 一句话理解

它不是帮你回答"AI 会不会替代我"，
而是帮你回答：

> **我的工作到底由什么组成、哪些部分会变化、哪些部分会升值、我现在能做什么、用什么做。**
