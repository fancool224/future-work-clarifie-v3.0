# 通用版标注标准 — S1-S6 受控词汇表

> 这是 Agent 内部标注工作颗粒度时的标准词汇表。每个标签都是封闭枚举值，Agent 只能从中选择。

---

## S1 工作性质标签

```
规则型 | 流程型 | 风险型 | 市场型 | 关系型 | 设计型 | 混合型
```

如果一个工作对象同时具有多种性质且不可拆分，标为"混合型"并列出主要叠加类型。

---

## S2 颗粒度字段标准

每个颗粒度必须包含以下 8 个字段。格式为表格行：

| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | 短文本 | 5-15 字，动宾结构（如"异常数据识别与通知"） |
| 定义 | 一句话 | 说清这个颗粒度做什么、产出什么 |
| 典型场景 | 短文本 | 在什么情况下触发 |
| 当前主要角色 | 角色名 | 现在主要由谁承担 |
| 是否高频 | 是/否 | 每天/每周发生 = 是 |
| 是否高价值 | 是/否 | 对核心目标有直接撬动 = 是 |
| 是否强依赖经验 | 是/否 | 新手 3 个月内无法独立 = 是 |
| 是否跨角色/跨部门 | 是/否 | 涉及 2+ 角色/部门协同 = 是 |

## S3 问题处理模式标签

```
规则判定型 | 风险识别型 | 归因诊断型 | 方案设计型 | 共识收敛型
```

每个标签的含义：
- **规则判定型**：寻求明确结论。有标准答案或可通过规则/数据计算得出。
- **风险识别型**：寻求风险等级、概率、趋势判断。结论只能是概率性的。
- **归因诊断型**：寻求原因解释。需要拆解因果链，定位根因。
- **方案设计型**：寻求可行动方案。需要从约束条件出发设计路径/机制。
- **共识收敛型**：寻求多角色判断对齐。核心是协调不同立场形成共同判断。

一个颗粒度可以有 1-2 个 S3 标签。

---

## S4 执行主体标签

```
自动接管 | AI主辅协同 | 人工主导，AI辅助 | 人工主导，AI仅弱辅助
```

判断标准：
- **自动接管**：AI 可以端到端完成，人仅做异常审核或抽样检查
- **AI主辅协同**：AI 完成第一层处理（初筛/初稿/初判/初排），人做升级判断
- **人工主导，AI辅助**：人做核心判断和决策，AI 提供信息检索/摘要/选项生成
- **人工主导，AI仅弱辅助**：AI 基本不介入核心环节，或仅做旁路辅助

---

## S5 价值增益标签

```
提效 | 提质 | 扩容 | 洞察 | 设计
```

每个标签的含义：
- **提效**：更快、更省人力、减少等待
- **提质**：更准、更稳、更少出错。高争议场景中也包含"组织接受度提升"
- **扩容**：覆盖更多对象/场景/频次/语种
- **洞察**：看见过去看不见的模式、关联、趋势
- **设计**：辅助生成和评估方案/机制/架构选项

一个颗粒度可以有 1-2 个 S5 标签。

---

## S6-A 能力标签

```
A:读取理解 | B:规则流程 | C:检索召回 | D:分类分级 | E:生成表达 | F:推理比较 | G:交互协同 | H:外部行动
```

每个代码对应的具体能力见 `framework-overview.md` S6-A 节。一个颗粒度通常标 2-5 个能力标签。

---

## S6-B 约束标签

| 维度 | 可选值 |
|------|--------|
| 技术成熟度 | 高 / 中 / 低 |
| 市场供给状态 | 充足 / 有限 / 空白 |
| 成本等级 | 低 / 中 / 高 |
| 启动主体 | 个人可启动 / 团队可启动 / 需要组织级投入 |
