# S1-S6 标注标准

## S1 工作性质识别
标签：规则型 / 流程型 / 风险型 / 市场型 / 关系型 / 设计型 / 混合型

## S2 工作颗粒度拆解
每个颗粒度至少包含：
- 名称
- 定义
- 典型场景
- 当前主要角色
- 是否高频
- 是否高价值
- 是否强依赖经验
- 是否跨角色/跨模块

## S3 问题处理模式
- 规则判定型：求明确结论
- 风险识别型：求风险等级、概率、趋势
- 归因诊断型：求原因解释、因果拆解
- 方案设计型：求可行动方案、机制设计
- 共识收敛型：求多角色判断对齐

## S4 执行主体重构
- 自动接管
- AI主辅协同
- 人工主导，AI辅助
- 人工主导，AI仅弱辅助

## S5 价值增益方式
- 提效
- 提质
- 扩容
- 洞察
- 设计

说明：
- “提质”默认指更稳、更准、更一致、错误更少。
- 在绩效、薪酬等高争议模块中，“提质”也可能包含“组织接受度提升”“争议减少”“结果更容易被管理者和员工接受”。

## S6 AI能力映射
### A 读取理解
文档读取 / 语义理解 / 关键词识别 / 结构化抽取

### B 规则流程
规则匹配 / 条件判断 / 流程状态识别 / 自动提醒升级

### C 检索召回
知识检索 / 相似案例召回 / 人才召回 / 历史事件追踪

### D 分类分级
分类 / 排序 / 风险评分 / 异常识别

### E 生成表达
文本生成 / 结构化总结 / 方案草案 / 多版本表达

### F 推理比较
归因推理 / 方案比较 / 约束下决策支持 / 多因素权衡

### G 交互协同
多轮追问 / 分歧识别 / 共识收敛 / 角色化表达

### H 外部行动
浏览器操作 / 系统调用 / 表格操作 / 消息触达
