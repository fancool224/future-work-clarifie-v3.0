# 未来工作厘定助手 - 核心框架总览

## 这是什么

这是一套给 Agent 使用的**内部判断框架**，不是给用户直接填写的问答清单。

它的作用是帮助 Agent 理解、拆解、判断一个 HR 相关“工作”在 AI 时代会如何变化。

这里的“工作”可以是：
- 一个完整模块
- 一个子模块
- 一个高阶颗粒度
- 一个跨模块关键议题
- 一组性质相近的工作集合

---

## 六层核心框架

### S1 工作性质识别
先判断当前被讨论的工作，本质上更像哪种工作系统：
- 规则型
- 流程型
- 风险型
- 市场型
- 关系型
- 设计型
- 混合型

### S2 工作颗粒度拆解
把一个大而模糊的工作对象拆成可判断、可推演、可比较的最小分析单位。

### S3 问题处理模式
判断这类问题本质上应该怎样被处理：
1. 规则判定型
2. 风险识别型
3. 归因诊断型
4. 方案设计型
5. 共识收敛型

### S4 执行主体重构
判断未来主要由谁来做：
1. 自动接管
2. AI主辅协同
3. 人工主导，AI辅助
4. 人工主导，AI仅弱辅助

### S5 价值增益方式
判断 AI 介入后的主要增益是什么：
1. 提效
2. 提质
3. 扩容
4. 洞察
5. 设计

### S6 AI介入方式 / 能力映射
判断 AI 靠什么能力介入：
- A 读取理解
- B 规则流程
- C 检索召回
- D 分类分级
- E 生成表达
- F 推理比较
- G 交互协同
- H 外部行动

---

## 使用顺序

1. 先做 S1（工作性质识别）
2. 再做 S2（颗粒度拆解）
3. 再做 S3（问题处理模式）
4. 再做 S4（执行主体重构）
5. 再做 S5（价值增益方式）
6. 最后做 S6（AI能力映射）

---

## 核心原则

- 不能直接按传统 HR 六大模块讨论未来，必须先拆颗粒度。
- 颗粒度拆解规则不能预设统一模板，必须先讨论工作性质。
- 不是所有问题都应该追求确定答案。
- 未来变化必须分成“执行主体变化”和“工作形态变化”两层来看。
- 最有价值的未来变化点，往往出现在跨模块关键颗粒度上。
- 某些模块（尤其绩效、薪酬）中的高阶判断，会强烈受组织博弈影响，不能被过度规则化。
- “提质”在部分模块里，不只是更准更稳，也包括让组织更容易接受结果、降低争议和沟通成本。

## 跨模块关键颗粒度

这是本框架中的一个正式概念。

定义：表面上看属于某个模块，但本质上必须由多个角色共同判断、共同收敛的高价值工作颗粒度。

典型例子：
- 招聘：用人需求校准与岗位供给方案共创
- TD：关键能力供给与继任配置设计
- ER：组织氛围感知与员工关系预防性干预
- 绩效：目标定义与对齐、校准与争议处理、结果应用中的高风险部分

这类颗粒度通常具备三个特征：
1. 不适合单模块关门处理
2. 未来变化幅度大、价值升高快
3. 往往是管理层最值得先共创的议题
