# 产品输入规范（v0.1）

## 最低可用输入
用户至少需要提供：
1. 本次讨论对象（一个模块 / 一个高阶颗粒度 / 一个跨模块议题）
2. 当前主要职责说明
3. 当前最耗时间的工作
4. 当前最难、最依赖经验的工作
5. 当前最想解决的问题
6. 本次参与角色

## 推荐输入文件

### 文件1：职责说明
格式：Word / PPT / Markdown / 文本
字段建议：
- 工作名称
- 工作目标
- 当前职责
- 当前组织分工
- 关键上下游
- 当前痛点

### 文件2：工作清单
格式：Excel / CSV / 表格文本
字段建议：
- 工作名称
- 工作说明
- 执行角色
- 频率
- 当前痛点
- 是否有明确规则
- 是否依赖经验
- 是否跨角色 / 跨模块

### 文件2-增强版：工作颗粒度拆解工具表
推荐直接使用：
- `references/general/hr-ai-work-inventory-template-v3.xlsx`

如果用户不知道从哪里开始，可以先填这张表。它本质上是 S2 工作颗粒度拆解层的核心工具表。

### 文件3：流程或典型案例
格式：SOP / Word / PPT / Excel / Markdown
字段建议：
- 当前流程
- 常见卡点
- 高风险场景
- 高频异常情况

## 输入原则
- 文件不全也可启动，先用最低输入包开始
- 有文件时优先做结构化抽取
- 文件只作为辅助，不替代共创讨论
